Arrays / Lists in Dart

Overview

  • Sequential storage with fast random access
  • Access: O(1) | Search: O(n) | Insert/Delete: O(n)
  • Costly insertions/deletions in the middle
Use Cases: Item lists, user data storage, displaying collections

Dart Implementation

void arrayListExample() { List<String> users = ['Alice', 'Bob', 'Charlie']; print(users[0]); users.add('David'); users.insert(1, 'Eve'); users.removeAt(2); users.remove('Bob'); bool hasAlice = users.contains('Alice'); for (var user in users) { print(user); } print(users.length); users.clear(); }

Linked Lists in Dart

Overview

  • Nodes with data and pointer to next node
  • Easy insert/delete at ends: O(1)
  • Slow random access: O(n)
Use Cases: Playlists, undo stacks, navigation history

Dart Implementation

class Node<T> { T data; Node<T>? next; Node(this.data); } class LinkedList<T> { Node<T>? head; void addFirst(T data) { Node<T> newNode = Node(data); newNode.next = head; head = newNode; } void addLast(T data) { if (head == null) { head = Node(data); return; } Node<T> current = head!; while (current.next != null) { current = current.next!; } current.next = Node(data); } void deleteFirst() { if (head != null) { head = head!.next; } } void display() { Node<T>? current = head; while (current != null) { print(current.data); current = current.next; } } }

Stacks & Queues in Dart

Stack (LIFO - Last In, First Out)

Use Cases: Undo operations, function calls, navigation history, back button
class Stack<T> { final List<T> _items = []; // Push - O(1) void push(T item) { _items.add(item); } // Pop - O(1) T? pop() { if (isEmpty()) return null; return _items.removeLast(); } // Peek - O(1) T? peek() { if (isEmpty()) return null; return _items.last; } bool isEmpty() => _items.isEmpty; int get size => _items.length; }

Queue (FIFO - First In, First Out)

Use Cases: Task scheduling, message handling, print queue
import 'dart:collection'; class TaskQueue<T> { final Queue<T> _queue = Queue(); // Enqueue - O(1) void enqueue(T item) { _queue.add(item); } // Dequeue - O(1) T? dequeue() { if (isEmpty()) return null; return _queue.removeFirst(); } T? peek() => _queue.isEmpty ? null : _queue.first; bool isEmpty() => _queue.isEmpty; int get size => _queue.length; }

Hash Tables (Hash Maps) in Dart

Overview

  • Key-value pairs with hashing function
  • O(1) average for search, insert, delete
  • Handles collisions automatically
Use Cases: Caching, quick lookups, user preferences, API response mapping

Dart Implementation

void hashMapExample() { // Basic Map (HashMap) Map<String, dynamic> userCache = {}; // Insert - O(1) userCache['user_123'] = { 'name': 'John Doe', 'age': 25, 'email': 'john@example.com' }; // Access - O(1) var user = userCache['user_123']; // Check existence - O(1) bool exists = userCache.containsKey('user_123'); // Delete - O(1) userCache.remove('user_123'); } // API Cache Example class ApiCache { final Map<String, dynamic> _cache = {}; void cacheResponse(String endpoint, dynamic data) { _cache[endpoint] = { 'data': data, 'timestamp': DateTime.now() }; } dynamic getCached(String endpoint) { if (!_cache.containsKey(endpoint)) return null; var cached = _cache[endpoint]; var age = DateTime.now().difference(cached['timestamp']); if (age.inMinutes < 5) { return cached['data']; } _cache.remove(endpoint); return null; } }

Trees & Graphs in Dart

Binary Search Tree

Use Cases: Search features, autocomplete, sorted data
class BinaryTreeNode<T extends Comparable> { T data; BinaryTreeNode<T>? left; BinaryTreeNode<T>? right; BinaryTreeNode(this.data); } class BinarySearchTree<T extends Comparable> { BinaryTreeNode<T>? root; void insert(T value) { root = _insertRec(root, value); } BinaryTreeNode<T> _insertRec(BinaryTreeNode<T>? node, T value) { if (node == null) { return BinaryTreeNode(value); } if (value.compareTo(node.data) < 0) { node.left = _insertRec(node.left, value); } else { node.right = _insertRec(node.right, value); } return node; } bool search(T value) { return _searchRec(root, value); } bool _searchRec(BinaryTreeNode<T>? node, T value) { if (node == null) return false; if (node.data == value) return true; if (value.compareTo(node.data) < 0) { return _searchRec(node.left, value); } return _searchRec(node.right, value); } }

Graph Structure

Use Cases: Social networks, maps, recommendations
class Graph<T> { final Map<T, List<T>> _adjacencyList = {}; void addVertex(T vertex) { _adjacencyList[vertex] ??= []; } void addEdge(T from, T to) { addVertex(from); addVertex(to); _adjacencyList[from]!.add(to); } List<T> getNeighbors(T vertex) { return _adjacencyList[vertex] ?? []; } }

Challenges in Mobile Development

‚ö†Ô∏è Key Challenges

  • Limited Memory: Constrained RAM (2-8GB)
  • Processing Power: Less powerful CPUs
  • Battery Life: Inefficient algorithms drain battery
  • Network Latency: Slow connections need caching

Solutions

1. Choose Right Data Structure

Use HashMap instead of List for lookups

2. Implement Caching

Store frequently accessed data in memory

3. Lazy Loading

Load data only when needed

4. Background Processing

Use queues for heavy operations
